# CLM User Stories

## Business User Stories

| Story ID | Title | User Story | Priority | Acceptance Criteria | Notes |
| --- | --- | --- | --- | --- | --- |
| BU-001 | View Dashboard | As a Business User, I want to view the CLM dashboard so that I can understand the status of my NDA and contract activity. | High | Dashboard is visible; NDA status counts display In Review, Approved, Awaiting Signature, Signed, and Archived; clicking a status shows only matching records. | Counts must match visible filtered records. |
| BU-002 | Request a New Contract | As a Business User, I want to start a new NDA request from the dashboard so that I can initiate a contract without emailing legal manually. | High | Request a New Contract opens a full-page workflow; Executive Viewer cannot start the flow; request flow includes all required steps. | New Contract Request must not be a sidebar item. |
| BU-003 | Complete Counterparty Details | As a Business User, I want to enter required counterparty and contract details so that legal has the information needed to review the NDA. | High | Required fields show stars; missing required fields produce validation errors; completed section turns green; incomplete section turns red after validation. | Required fields include company, counterparty, email, addresses, dates, term, project, purpose, jurisdiction, and governing law. |
| BU-004 | Calculate Agreement Period | As a Business User, I want the system to calculate end date or term time so that I do not manually calculate NDA duration. | High | Start Date plus Term Time calculates End Date; Start Date plus End Date calculates Term Time; exact durations display as text such as "1 year 3 months 12 days". | Do not show "Custom Time". |
| BU-005 | Select NDA Template | As a Business User, I want to select an NDA template during intake so that my draft is generated from the correct template. | High | Template selection is available; quick templates include Standard NDA, Mutual NDA, and Employee / Contractor NDA; more templates are searchable in dropdown; selected template is stored with request. | Template preview is optional. |
| BU-006 | Review Placeholder Mapping | As a Business User, I want to see which placeholders will be populated so that I can correct missing values before generating a draft. | Medium | Detected placeholders are shown; mapped and missing statuses are visible; missing required placeholder values trigger warning before final document generation. | Preview must not modify original template. |
| BU-007 | Answer Risk Questions | As a Business User, I want to answer NDA risk and data questions so that the system can recommend appropriate legal review. | High | Risk questions include competitor risk, M&A/JV, international business, new IP, similar products, third-party information, pricing/strategy/roadmap, PII, employee data, customer data, large data files, and cross-border transfer. | Answers feed risk scoring. |
| BU-008 | Run Risk Review | As a Business User, I want to run a risk review so that I can understand whether my NDA needs additional review. | High | Risk Review shows score, level, flags, explanation, recommended template, approval workflow, clause recommendations, placeholder status, and document status. | Business User cannot approve or send to sign. |
| BU-009 | Save Draft | As a Business User, I want to save my NDA request as a draft so that I can finish it later. | High | Save as Draft persists current form state; toast says "Draft saved successfully"; draft reopens with details prefilled; draft is not set to In Review until submitted. | Store draft locally for demo persistence if backend is unavailable. |
| BU-010 | Submit NDA Request | As a Business User, I want to submit my NDA request so that legal can review it. | High | Submit validates required fields; creates a repository record; sets status to In Review; updates dashboard; shows success message; creates audit event Request submitted. | No separate Tasks section is created. |
| BU-011 | View Repository | As a Business User, I want to view repository records so that I can track my NDAs and related documents. | High | Repository is visible; all documents are listed; user can view records and download permitted documents; restricted actions are blocked. | Repository visibility is not role-limited. |
| BU-012 | Download Generated Documents | As a Business User, I want to download generated DOCX and PDF files so that I can review or share the NDA as permitted. | Medium | Download buttons work when placeholders are complete; missing placeholders show error toast; audit entries are created for downloads. | Signed records should download stored signed files. |
| BU-013 | Use AI Search | As a Business User, I want to ask questions about NDA records so that I can quickly find status, expiry, risk, and counterparty details. | Medium | AI Search is visible; user can select an NDA; exact field questions return exact answers; missing information returns the specified missing-data message. | Rename AI Tools to AI Search everywhere. |
| BU-014 | Receive Permission Feedback | As a Business User, I want clear feedback when I try an action I cannot perform so that I understand access limits. | High | Approve, reject, send to sign, delete, and admin settings are blocked; message says "You do not have permission to perform this activity." | Applies to all restricted actions. |

## Legal Reviewer Stories

| Story ID | Title | User Story | Priority | Acceptance Criteria | Notes |
| --- | --- | --- | --- | --- | --- |
| LR-001 | View Review Queue | As a Legal Reviewer, I want to see NDAs in review so that I can prioritize legal work. | High | In Review records are visible from dashboard or repository; status, risk level, counterparty, template, expiry date, and owner are visible. | Legal Review Queue may be a filtered view or section. |
| LR-002 | Preview NDA Document | As a Legal Reviewer, I want to preview the generated NDA so that I can review the contract before approving it. | High | Preview document action appears under Legal Reviewer section; preview shows populated NDA details and template metadata. | Must not be grouped under Tasks. |
| LR-003 | Edit NDA Details | As a Legal Reviewer, I want to edit an existing NDA request with all details prefilled so that I can correct or complete the record. | High | Edit opens existing NDA; counterparty details and all workflow fields are prefilled; saving updates same record; audit entry is created. | Must not start from scratch. |
| LR-004 | Review Risk Explanation | As a Legal Reviewer, I want to see risk flags and explanations so that I can make an informed approval decision. | High | Risk score, level, selected flags, explanations, and recommendations display; PII and competitor risk use required explanation text. | Risk should be deterministic. |
| LR-005 | Approve NDA | As a Legal Reviewer, I want to approve an NDA so that it can proceed to signature. | High | Approve is enabled for In Review records; status changes to Approved; success toast appears; audit entry is created; unauthorized roles are blocked. | Allowed for Legal Reviewer and Admin. |
| LR-006 | Reject NDA | As a Legal Reviewer, I want to reject an NDA so that unsuitable requests are removed from active processing. | High | Reject moves record to Archived or clearly marks rejection through audit/review notes; success toast appears; audit entry is created. | Status must remain within allowed status set. |
| LR-007 | Send NDA to Sign | As a Legal Reviewer, I want to send an approved NDA for signature so that the counterparty can execute it. | High | Send to Sign is enabled only when status is Approved; status changes to Awaiting Signature; signing link is generated; audit entry is created; success message appears. | Uses counterparty email from intake. |
| LR-008 | Validate Final Document | As a Legal Reviewer, I want to verify placeholder completion before download or signature so that incomplete NDAs are not sent out. | High | Missing required placeholders block final document generation or download; visible status shows mapped/missing placeholders. | Prevents incomplete legal documents. |
| LR-009 | Download DOCX and PDF | As a Legal Reviewer, I want to download generated NDA files so that I can review and share the correct document format. | Medium | DOCX and PDF downloads work; filenames are meaningful; audit entries are created. | PDF and DOCX contain same populated values. |
| LR-010 | View Audit Trail | As a Legal Reviewer, I want to see audit history so that I can understand how an NDA moved through the lifecycle. | Medium | Audit entries include draft saved, submitted, template selected, risk review, document generated, downloads, approval, rejection, sent to signature, signed, archived, deleted, and export events where applicable. | Events include actor and timestamp. |
| LR-011 | Use Templates | As a Legal Reviewer, I want to view template details so that I can confirm the correct NDA template is used. | Medium | Templates page visible; Legal Reviewer can preview and use templates; template creation remains Admin-only. | Legal Reviewer cannot manage admin settings. |
| LR-012 | Use AI Search for Review | As a Legal Reviewer, I want AI Search to answer questions about selected NDAs so that I can review records faster. | Medium | AI Search returns exact expiry, status, party, purpose, risk, signature, template, email, and address details when available. | Search selected NDA first. |

## Admin Stories

| Story ID | Title | User Story | Priority | Acceptance Criteria | Notes |
| --- | --- | --- | --- | --- | --- |
| AD-001 | Manage All Actions | As an Admin, I want access to all actions so that I can configure and operate the CLM platform. | High | Admin can request, edit, approve, reject, send to sign, delete, export, manage users, templates, parties, rules, and settings. | Admin bypasses normal restricted action limits. |
| AD-002 | Manage Users | As an Admin, I want to add and manage users so that roles and access can be controlled. | High | Admin can add users with Title, First Name, Last Name, Email, and Role; users table displays professional demo names; non-admins cannot access management. | Settings > Users. |
| AD-003 | Manage Templates | As an Admin, I want to create and manage NDA templates so that the business can use approved language. | High | Admin can add new NDA templates; save templates; manage placeholders; newly created templates appear in request workflow. | Template metadata must include name, type, jurisdiction, version, date, status, placeholders. |
| AD-004 | Preview Template Placeholders | As an Admin, I want to preview template placeholders so that I can confirm mapping before use. | High | Preview shows original template; placeholders are highlighted; side panel shows detected placeholders and mapped/missing status. | Original template remains unchanged. |
| AD-005 | Manage Parties | As an Admin, I want to add, edit, search, and export counterparties so that reusable party data is available. | Medium | Parties page is Admin-only; add/edit/search/export works; dropdowns are readable in dark and white themes. | Export creates CSV of visible data. |
| AD-006 | Manage Rules | As an Admin, I want to configure routing rules so that NDA requests are assigned consistently. | High | Rules page is Admin-only; Admin can add routing rules; rules show during NDA creation; examples include high risk, PII, international business, large data files, and low-risk standard NDA. | Rules inform approval workflow recommendations. |
| AD-007 | Delete Document | As an Admin, I want to delete or archive documents so that records can be removed from active use when appropriate. | Medium | Delete document appears in row action menu; only Admin can delete; audit entry Deleted is created; non-admins see permission message. | Deletion behavior may archive or remove from active list based on implementation. |
| AD-008 | Export Data Tables | As an Admin, I want export buttons to generate CSV files so that reporting data can be shared. | Medium | Repository, Parties, Rules, Templates, and other visible tables export currently visible rows and columns; audit event Export completed is created. | File names should be meaningful. |
| AD-009 | Configure Theme | As an Admin, I want to switch between dark and white theme so that the UI supports user preference and readability. | Medium | Theme toggle works; dropdowns remain readable; application state reflects selected theme. | Available to all users if implemented as preference. |
| AD-010 | Verify Role Restrictions | As an Admin, I want role restrictions to be predictable so that access control can be trusted. | High | Business User cannot approve/send to sign; Legal Reviewer cannot access Admin-only settings; Executive Viewer is view-only; unauthorized actions show required permission message. | Must be included in regression checklist. |
| AD-011 | Manage Signed Documents | As an Admin, I want signed documents stored against NDA records so that the repository is complete. | High | Signed PDF and DOCX are stored after signature; status changes to Signed only after storage; repository shows signed files. | Applies to e-sign workflow. |
| AD-012 | View Full Audit Trail | As an Admin, I want a complete audit trail so that compliance and troubleshooting needs are supported. | Medium | All required audit events are captured with actor, role, timestamp, and record reference. | Useful for QA and stakeholder demos. |

## Executive Viewer Stories

| Story ID | Title | User Story | Priority | Acceptance Criteria | Notes |
| --- | --- | --- | --- | --- | --- |
| EV-001 | View Dashboard Summary | As an Executive Viewer, I want to view status summaries so that I can understand contract operations performance. | High | Dashboard is visible; status counts show allowed statuses; record type overview is visible; no create actions are available. | View-only access. |
| EV-002 | View Repository Records | As an Executive Viewer, I want to view repository records so that I can inspect contract status and details. | High | Repository is visible; all documents are visible; view action opens document section; edit/delete/approve/send actions are blocked or hidden. | Repository visibility applies to all roles. |
| EV-003 | View Signed Documents | As an Executive Viewer, I want to view signed NDA documents so that I can confirm execution status. | Medium | Signed records show status Signed; signed PDF/DOCX metadata is visible; view full document is available where allowed. | Download policy may be configured, but no modification allowed. |
| EV-004 | View Expiring NDAs | As an Executive Viewer, I want to see expiring NDA indicators so that I can understand upcoming renewal exposure. | Medium | NDAs expiring within 30 days are identifiable; renew action is not available to Executive Viewer. | Renewals are not dashboard summary tabs. |
| EV-005 | Use Read-Only Theme Preference | As an Executive Viewer, I want to switch theme so that I can view CLM comfortably. | Low | Theme switching works without granting edit privileges. | Settings access may be limited to preference only. |
| EV-006 | Receive Permission Feedback | As an Executive Viewer, I want clear permission feedback so that I know why actions are unavailable. | High | Creating, editing, approving, sending, deleting, managing users, parties, rules, and templates are blocked; required permission message appears if attempted. | Executive Viewer must be view-only across application. |

## Counterparty Signer Stories

| Story ID | Title | User Story | Priority | Acceptance Criteria | Notes |
| --- | --- | --- | --- | --- | --- |
| CS-001 | Open Signing Link | As a Counterparty Signer, I want to open a signing link from email so that I can access the NDA without logging into CLM. | High | Link is clickable and accessible; route opens `app/sign/[token]/page.jsx`; token maps to the correct NDA; link does not expire immediately. | Must work for any recipient email. |
| CS-002 | Preview NDA Before Signing | As a Counterparty Signer, I want to preview the full NDA so that I can review the contract before signing. | High | Signing page shows complete NDA content; generated values are populated; signature action remains enabled even if preview/download is skipped. | Preview is useful but not mandatory. |
| CS-003 | Download Before Signing | As a Counterparty Signer, I want to download the NDA before signing so that I can keep a copy for review. | Medium | Download option is available before signing; document matches generated final NDA. | Download should not block signing. |
| CS-004 | Complete E-Signature | As a Counterparty Signer, I want to complete a proper e-signature action so that the NDA can be executed. | High | Sign and Return captures signature; signature is embedded into final document; signed PDF and signed DOCX are generated and stored. | More than typing a name should be represented where practical. |
| CS-005 | Confirm Signed Submission | As a Counterparty Signer, I want confirmation after signing so that I know the NDA was returned successfully. | High | Confirmation page appears after Sign and Return; no Return to CLM Dashboard link is shown; signed document storage succeeds before status changes to Signed. | External signer should not see internal app navigation. |
| CS-006 | Download Signed NDA | As a Counterparty Signer, I want to download the signed NDA immediately after signing so that I have a final executed copy. | High | Confirmation page shows Download Signed NDA; signed PDF and signed DOCX downloads are available when stored. | Status must be Signed after storage. |

## Cross-Role Stories

| Story ID | Title | User Story | Priority | Acceptance Criteria | Notes |
| --- | --- | --- | --- | --- | --- |
| CR-001 | Accurate Status Flow | As any CLM user, I want statuses to update correctly so that contract lifecycle state is reliable. | High | Flow uses only In Review, Approved, Awaiting Signature, Signed, and Archived; transitions follow role permissions. | Dashboard and repository must stay in sync. |
| CR-002 | Audit Lifecycle Events | As any authorized CLM user, I want lifecycle actions audited so that activity history is transparent. | High | Required audit events are captured with actor, role, timestamp, event, and record reference. | Applies across workflows. |
| CR-003 | Consistent CSV Export | As any user with table access, I want CSV exports to reflect visible data so that I can report accurately. | Medium | Export includes currently visible rows and columns; success toast appears; audit entry created. | Export permissions follow table visibility. |
| CR-004 | Accurate AI Answers | As any AI Search user, I want exact answers from contract records so that I can trust CLM search results. | High | Field-specific questions return exact data; selected NDA is searched first; repository-wide search works when no NDA selected; missing-data message appears only when data is truly absent. | Local intent matcher can be used initially. |
| CR-005 | Document Integrity | As any authorized user, I want generated and signed documents to match the selected template so that CLM produces reliable legal documents. | High | Placeholders are replaced correctly; required placeholders block generation when missing; PDF and DOCX contain same populated values; signed files are stored and downloaded from repository. | Key QA area. |
